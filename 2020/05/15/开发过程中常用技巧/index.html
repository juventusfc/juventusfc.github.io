开发过程中常用技巧

Search in Chrome Dev Tool

当需要查找 Network 里某个代码片段在哪个文件中时，如果一个个文件去找就太麻烦了。可以在 Chrome Dev Tool 中按Ctrl + Shift + F快速查找代码片段

Can I Use 查询浏览器是否支持

使用Can-I-Use，能查询各个浏览器对 js/css 的支持程度。

多个版本 node 之间的切换

使用 nvm 来管理多个 node 版本。nvm list指令用于查看本机安装了哪些版本 node。

查看全局安装的 npm 包

使用npm list -g --depth=0

VSCODE 实用快捷键

快速生成 html5：! + Tab + Tab

VSCODE 同步配置

使用 Setting Sync 插件同步 VSCODE 的各个插件。

  • GitHub Token: *f8d7f89d6e67d6a2fefb39d30e1f79a1ff17c9c3**
  • GitHub Gist: *d54bacfef33d7af262c8df43db3078df**
  • GitHub Gist Type: Secret

在同步时可能发生“Invalid / Expired GitHub Token. ”错误。解决方法如下：

  1. VSCode 中输入命令(Ctrl + Shift + P)：Sync:Advanced Options，然后选择 Sync:Edit Extension Local Settings，编辑 syncLocalSettings.json 这个配置文件。这个文件中有一项 token 没有设置（或者是配置的就是以前的过期的 token），这里就需要设置为 Token Id。
  2. 生成新的 Token。
  3. syncLocalSettings.json 中的 token 使用新产生的 token。

BootStrap 检测工具，检测有没有正确使用 BootStrap

BootLint

设计 API

构建数据模型设计

使用docson来构建数据模型并可视化。使用方法：

  1. 下载docson源码
  2. 使用docson -d ./examples开启服务，打开自己构建的模型文件夹
  3. 各方参与人员可以在线查看

API 接口的设计

使用swagger editor。使用方法：

  1. 下载swagger editor源码
  2. 执行npm start。注意，node 版本需要为 6.x

查看电脑的 cpu 和核心数

  1. Open Task Manager
  2. Open Resource Monitor

Windows 中并排两个窗口

如果要将窗口 1 和窗口 2 并排查看，可以将窗口 1 移至最右边，然后会自动出现所有的任务缩略图，选择要并排的窗口 2。
