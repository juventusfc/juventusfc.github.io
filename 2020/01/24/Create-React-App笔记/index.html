browserslist

browserslist 表示该项目支持的浏览器。browserslist 不能单独使用，使用场景主要有：

  • Babel 工具根据 browserslist 的配置来转换 JavaScript 代码。
  • Autoprefixer 工具根据根据 browserslist 的配置，自动生成对应浏览器的前缀。

具体可参考 browserlist-good-idea。

  1. 在执行 npm run start 时，IE11 里面打开时白屏的；
  2. 但是执行 npm run build 然后放到静态服务器上时， IE11 里面打开是正常的。

react-app-polyfill

react-app-polyfill 是一款 polyfill 工具。

在 IE9 中使用 Create React App 生成的代码时，需要配置：

  1. src/index.js 中增加 react-app-polyfill。

  2. 修改 package.json 中的 browserslist，使项目支持 IE9。

ESLint

当代码有 Lint Error 时，VSCode 中有提示

Create React App 中默认赔了 ESLint，但是该 ESLint 只有一些基本配置，基本没啥用。

如果要自己添加规则，请执行以下步骤：

  1. VSCode 中添加 ESLint 插件

  2. 修改 package.json 中的 eslintConfig 配置，增加规则

请注意，以上步骤只会在 VSCode 中增加 Lint 信息，命令行或浏览器中不包含 Lint 信息。也就是说，VSCode 中的 ESLint 插件会去读取 package.json 中的 eslintConfig 字段。

当代码有 Lint Error 时，不允许 start 或 build

  1. 修改 package.json 中的 eslintConfig 配置，增加规则

  2. 增加 .env 文件

  3. 代码中，故意使用 var a = 100;

  4. 执行 npm run start 或 npm run build，发现已经不能 start/build 了

也就是说， .env 文件会在 start 或 build 时，被读取，然后根据其中的配置，注入到生成的代码中。

环境变量

设置环境变量的方式有两种：

  1. 在 shell 中设置。这种方式主要用于调试时。

  2. 是设置在配置文件中，比如 .env。

如果两种方式设置了同一个环境变量，以 shell 中的为准。

环境变量会在编译的时候编译进生成的 JavaScript 中。

Test

  • Jest：提供测试框架。主要用于冒烟测试。包含了 expect() (Chai 的功能）、jest.fn() (Sinon 的功能)。
  • Enzyme：主要用于 Component 的测试。
  • react-testing-library:类似于 Enzyme。
  • Chai：断言库。
  • Sinon：Mock 库。

常用方案 1：
Jest + Enzyme

常用方案 2：
Jest + react-testing-library + jest-dom
