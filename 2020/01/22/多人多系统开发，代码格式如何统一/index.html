多人多系统开发，代码格式如何统一？

发表于 2020-01-22 更新于 2020-01-27 分类于 技术
Windows 平台和类 Unix 平台下，前端代码的统一开发，代码格式如何统一？

多人协作越来越是开源项目的开发方式。参与开发的人员会面临如下两个问题：

  1. 使用不同电脑操作系统
  2. 使用不同代码编辑器或 IDE

为了使代码格式统一，需要使用好 4 个工具。

Git

如果你使用 Windows 开发，由于 Windows 和类 Unix 系统对换行符的解释不同，所以需要执行以下步骤：

  1. 执行命令 git config --global core.autocrlf input。一般来说，代码库中的代码都是使用 lf 的。

    上图中，repo 指远程仓库。 core.autocrlf 的默认值是 true。执行 git config --list | grep autocrlf 可以查看当前的 core.autocrlf 的值。

  2. 克隆代码

EditorConfig

EditorConfig 的主要作用是当作项目的编程规范。有一些编辑器（如 Visual Studio）已经内置了这个功能，有些编辑器（如 Visual Studio Code）需要自己手动安装扩展。

一般是在项目根目录中创建一个 .editorconfig 文件，然后使用。

ESLint

ESLint 是 JavaScript linter，主要功能有：

  1. 制定代码规则。如不能有定义了但并未使用的变量等。
  2. 格式化代码。ESLint 提供了 fix 功能，能格式化代码。

在实际项目中，我们只会使用 ESLint 提供的第一种功能。

Prettier

Prettier 主要用于代码的自动格式化。在格式化代码这个功能上，比 ESLint 优秀。

ESLint + Husky + lint-staged + Prettier

现在，社区里常用的方案时使用 ESLint + Husky + lint-staged + Prettier 来自动格式化代码。自动格式化代码主要有两种方案：

  1. 在文件保存时自动格式化。
  2. 在文件 commit 时自动格式化。更进一步的，可以设置 pre-commit 和 pre-push，强制代码的统一。

由于每个人的编辑器和其中的设置都不一样，所以我推荐使用 Facebook 采用的第二种方式。具体流程为：

  1. ESLint 提供了 Lint 功能。这一步，建议打开 VSCode 中的 ESLint 插件，根据 Lint 提示更改。
  2. Husky 提供了 git 的钩子功能。
  3. lint-staged 提供了对 git staged 中的文件的修改功能。
  4. Prettier 提供了代码自动格式化功能。在这一步，也有人采用 ESLint 中的格式化功能。

具体可参考 formatting-code-automatically。
