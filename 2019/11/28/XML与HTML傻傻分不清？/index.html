XML 与 HTML 傻傻分不清？

发表于 2019-11-28 更新于 2019-11-29 分类于 技术
XML 与 HTML 的恩怨纠葛

XML 和 HTML 在开发中经常会碰到。两者之间有什么关联和区别呢？根据开发世界的发展历程，我们可以试着来理解一下。

Content Structure 和 Format 是早期开发需要解决的主要问题

早期开发过程中，当面对处理 Document 问题时，有三个问题是绕不开的：

  1. Content
    Document 的内容。信息主要包含在 Content 中。
  2. Structure
    Document 的结构。就像写文章，文章是有结构的，比如标题/章节等。
  3. Format
    Document 的展示形式。

XML、HTML 是为了解决实际问题而存在的

为了解决上述三个问题，GML(Generalized Markup Language)出现了。( 搞笑的是，GML 也是三位创造者 LastName 的简写。Charles Goldfarb, Edward Mosher and Raymond Lorie)

GML 的解决方法是增加 Tag。具体做法是将 Content 包含在 Tag 中，将 Structure 和 Format 放在 Tag 上。在此基础上，GML 还进一步标准化为SGML( standardized version of GML)。

但是，问题又出现了。SGML 太严格了，

  1. 开发者舍弃了一部分 SGML 中的 Structure 功能，开发出了 HTML
  2. 开发者舍弃了一部分 SGML 中的 Format 功能，开发出了 XML

现在好了，开发者可以同时使用 XML 和 HTML 来实现自己的目的，XML 用于提供 Content 和 Structure，HTML 提供 Content 和 Format。随着开发水平的进步，XML 也一步步演化为类似于 database 的存在。同时，更进一步的，当开发者需要 database 里的数据时，XML 承担了传输数据的任务。

是不是让你想起了 JSON？

没错，JSON(JavaScript Object Notation) 也提供了类似 XML 的传输数据任务。由名字就可以看出来，JSON 的发展主要受益于 JavaScript 的发展。
